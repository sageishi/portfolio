Concept

(架空サイト)
カフェと花屋が融合した
新しい感覚のお店。

プロジェクトのテーマ:
「雑踏から一歩踏み込めば
目から花、花から香り、
一時のあなたのカフェタイムから
静けさをお届けします」

トーン&マナー:
穏やか・ナチュラル
・優しい 深い・静か

完了後の感想介

初めて一から(Hearing, Creative Brief等)スタートしてウェブサイトを作成した。
今まで、他人の作品をトレースしたり、自分の思いつきでいきなりコーディングから始める事しかなく。
又は、他人が作ったサイトのデバッグ作業のみしていなく。あまりプロセスの大切さに重みを感じて
いなかったです。今回、自分で一から始める事でその重みを実感しました。
ディレクターがクライアントとのやり取りで決めたワイヤフレーム、
そのワイヤフレームの元にデザインを行うデザイナーの意図を与する事によって、
コーダーの能力が発揮される事を体験した。
今後、この初心の心を忘れずにコーダーとして成長して行きたいと思いました。

    jQueryのプラグインはあえて初めて使用するものにチャレンジしたが、裏で動作している 箇所を把握する前に組み合わせた。結果として、互いにバッティングする事になった。改めて、個々で一度動作を確認してから組み合わせる事により問題点を把握できた。次回は、 初めて使用するプラグインに対しては、一度、動作確認をしてから実装することを学んだ。 問題解決に対しての有効な手段・デバッグ方法を経験したので、今後活かしての作業に活 かして行きたい。

  • 06サイトのアップロード&動作確認

    サーバーにデータをアップロード、デバッグ作業

    所有時間: 5時間

    詳細を表示

    サーバーにデータをアップロード時にスマートフォンでの一部jQueryプラグインを使用したタッチスクロールが機能していない事に発覚。プラグインが最新ブラウザーに対応していない (ここ数年更新されていない)のが分かり、自分でプラグインのjsファイルを検索、問題箇所を修正。

  • 07サイトの最適化

    Mobile-Friendly Test、PageSpeed Insightsを利用して、 Webサイトの軽量化・最適化作業

    所有時間: 9時間

    詳細を表示

    GoogleのMobile-Friendly Testを使用してOKでしたが、複数回テストしていたら、何回か初期時に正しく表示されない箇所があった。修正を複数回行って、常時問題無く表示されるようにしました。

    PageSpeed Insightsを利用して、サイトのロード性能を確認した。最初は23点であり、下記のお勧め改善箇所などを修正を行った。お勧め以外の箇所も自分が思いつくかぎりに修正を行った。結果、モバイルサイトでは74点までに改善する事に成功した。

    上記と同じく、今度はデスクトップ画面でのロード性能を向上をした。最初は35点で始まり、お勧め改善箇所などを修正を行った。最初のSVGアニメーションなども、時間を短縮などしました。結果、94点に改善ができ、期待以上の結果を出す事に成功した。

  • 08SEO等の検索対応

    Google Search Consoleで検索上位表示の作業

    所有時間: 7時間

    詳細を表示

    サイトマップ作成時にリンクに複数エラーがある事が表示され、jQueryプラグインの記述方法に問題があるのを発覚。jsファイルを修正して、文章構造を直した。その後、Google Search Consoleにサイトを登録。登録一週間以内に上位表示に成功した。

  • サイト用のバナー制作

    詳細を表示

    Photoshopでサイト用のバナーを2サイズで作成。トーン&マナーを意識して、限られたスペースの中で表現してみました。300x250のバナーではキーワード選定時に一番検索されていた施設の設備をアイコンで表示しました。高解像度ディスプレイに対応する為にPhotoshopの「画像 アセット生成」機能を利用して2倍までの画像を出力した。
