完了後の感想介

Wordpressは使いやすく、誰でもブログを始められるイメージを持っていました。
しかし、授業でWordpressを学んでいくと、既に出来ているテンプレート(テーマ)を使用するだけで、
ひたすらブログを書くのが目的であれば、使いやすいと言われている事を理解しました。
ただし、テーマを個性あるカスタム化するには基本のhtml・cssだけではなく、 phpやデータベース、
あとはWordpressの独自の記述を理解していないと文章構造等が正しく表示されない事が
良く理解できました。今後、Wordpress等のCMSが必需になっていくので、
基本をしっかり身につけて様々なCMS等に対応できるように努力して行きたいと思いました。

一つのphpファイルで投稿リスト・投稿のシングルページ・
各自のモバイルへのレスポンシブデザイン

  1. the_excerpt()を使用したが、シングルページで投稿を表示した場合、改行やpタグが別々だったのが一つにまとまってしまい、表示が崩れてしまった
  2. カテゴリー名を文字だけでは無く、アイコンを表示したかった。ただし、投稿一覧のページとシングルページではWordpressの記述が違った
  3. dynamic_sidebar()をnav替りに使用したが、表示出来ない項目があった
  1. the_excerpt()ではなくthe_content()を使用して、投稿記事にはhtmlで内容を記載する事により、レイアウトをコントロールする事ができた
  2. Googleで検索してする事によって、シングルページではget_queried_object()、投稿ページはget_the_category()を使用するのが分かった
  3. custom htmlが見つかり、直接htmlを記入。表示はcssで修正を行った

Wordpressはプラグインが豊富にあるが、プラットフォームが常時アップデートされている為、プラグインが非対応になる可能性が高いと分かりました。テーマも更新されば、カスタムした部分がその都度消えてしまう。可能な限り自分でテーマを作り、プラグインに便りにし過ぎずに自分でfunctionなどを作る方が長期的に良いと実感しました。Wordpressにはまだまだ使用していない機能が沢山あるので、一個一個使えこなして行きたと思います。

is_category()の中でフィルターされているカテゴリーを引用するにはget_queried_object()が必要と分かりました。Googleで検索して、Objectのアローファンクションで呼び出すデータ名がterm_idと分かりました。後はテーマのディレクトリーの呼び出しのファンクションを利用して、画像のsrcに記述する。

最初はtaxonomy等のプラグインを使用してアイコンの表示を試みたけど、アイコンの表示はバックエンドの管理者ページでしか表示が出来なかった。Googleで検索してみたら、カテゴリーのIDを利用してphpで表示できる方法を学びました。上記のis_category()と比べて、get_the_category()の記述の方が分かりやすかったが。一つの投稿記事に複数の表示が必要な為、foreach()を使用して複数の表示ができた。
